Características

Características

Construido para la privacidad.
Diseñado para la velocidad.

Cada característica fue diseñada con un propósito: darte un navegador que trabaja para ti, no para los anunciantes.

Motor Rust Propio

Pipeline completo implementado en Rust puro: HTML → Tokenizer → DOM → CSS Cascade → Layout → Display List → FluxSoftRenderer → píxeles. El motor existe y funciona — su integración como pantalla principal está en desarrollo activo.

En construcción — integración en progreso

Búsqueda Privada

Motor de búsqueda propio con SearXNG self-hosted y re-ranking BM25. Sin Google, sin Bing, sin rastreo. Tus búsquedas nunca salen de tu máquina.

SearXNG + BM25

Bloqueo de Anuncios

3 capas de bloqueo: red (DNS/HTTP), DOM (inyección de selectores CSS), y YouTube específico con intercepción del reproductor. Sin excepciones pagadas.

YouTube sin anuncios incluido

100% Local-first

Historial, favoritos, notas y configuración almacenados en SQLite local. Ningún dato sale del dispositivo. Sin cuentas, sin sincronización en la nube, sin servidores.

SQLite · Tus datos son tuyos

Sin Telemetría

Cero datos enviados a servidores externos. Sin crash reports automáticos, sin métricas de uso, sin "mejora de servicios". El código es verificable — no hay telemetría oculta.

Código abierto y verificable

HTTPS + CSP

Upgrade automático de HTTP a HTTPS con HSTS enforcement. Content-Security-Policy aplicada en cada página. Sandbox de JavaScript con permisos explícitos.

Seguridad multicapa

Rendimiento

Diseñado para ser eficiente.

Flux prioriza la eficiencia de memoria. Actualmente usa WebView2 para el renderizado de páginas, mientras el motor propio en Rust se finaliza.

Uso aproximado de RAM con sesión básica (medición propia, puede variar según contenido y sistema):

F
Flux Browser ~180 MB
Google Chrome ~500–900 MB
Mozilla Firefox ~300–600 MB
Microsoft Edge ~400–750 MB

Los otros navegadores son proyectos maduros y bien optimizados. Estas cifras son orientativas — cada uno tiene sus fortalezas. Flux enfoca su diferencia en privacidad y datos locales, no en competencia directa.

El Creador

Hecho por una sola persona.

Flux no es el producto de una empresa. Es la visión de un desarrollador.

GZ

Gabriel Zapata

Creador y Desarrollador Principal · Flux Browser

Construí Flux como un proyecto personal para entender cómo funciona un navegador por dentro — desde el tokenizer hasta el renderizado de píxeles — y al mismo tiempo poner la privacidad del usuario en el centro. No es una crítica a los otros navegadores; hay proyectos increíbles ahí afuera. Flux es mi manera de aprender haciendo.

Cada componente — el tokenizer zero-copy, el DOM arena, el CSS cascade, el motor de búsqueda BM25, el bloqueador de anuncios — fue construido con un propósito claro: que los datos del usuario se queden en su máquina, siempre.
